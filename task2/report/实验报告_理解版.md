## 1. 任务理解

Track1 要求在已经给定基础矩阵分解模型的前提下，实现一个增量 SVD 模型。评测系统会先提供预训练好的用户矩阵 `P`、物品矩阵 `Q` 和全局均值 `global_mean`，再把新增评分数据按批次传入 `update()`。我们不能重新训练全量 MovieLens 20M 数据，只能利用新增批次对模型做增量修正。

需要实现的核心接口是：

```python
class IncrementalSVD:
    def __init__(self, num_users, num_items, latent_dim, global_mean): ...
    def load_base_model(self, user_matrix, item_matrix): ...
    def update(self, incremental_batch): ...
    def predict(self, user_id, item_id): ...
```

其中 `update()` 会被计时，因此算法不能只追求 RMSE，也要控制运行时间。最终有效提交要求新 RMSE 小于基础模型 RMSE，并且改进幅度超过 `0.001`。

## 2. 基础模型

基础 SVD 的预测形式为：

```text
pred(u, i) = global_mean + P[u] dot Q[i]
```

其中：

- `P[u]` 是用户 `u` 的潜在向量。
- `Q[i]` 是物品 `i` 的潜在向量。
- `latent_dim` 默认为 1024，向量维度较高。

如果用户或物品 ID 越界，按照作业说明返回 `global_mean`。预测结果需要裁剪到评分范围 `[0.5, 5.0]`。

## 3. 增量更新算法

本次实现采用分块 mini-batch SGD。对一条新增评分 `(u, i, r)`，先计算当前预测误差：

```text
err = r - pred(u, i)
```

再用如下规则更新用户向量和物品向量：

```text
P[u] <- P[u] + lr * (err * Q[i] - reg * P[u])
Q[i] <- Q[i] + lr * (err * P[u] - reg * Q[i])
```

代码实现中使用的是批量矩阵操作，不是一条一条 Python 循环。对一个 chunk 内的样本，先取出对应的 `P[u]` 和 `Q[i]` 快照，计算所有样本的预测值与误差，再一次性构造增量并用 `np.add.at()` 累加回原矩阵。

这样做的原因是：

1. Python 层循环 10 万条样本会很慢。
2. `latent_dim=1024`，每条评分都要做 1024 维点积和更新，适合交给 NumPy 底层向量化执行。
3. 一个批次中同一用户或物品可能重复出现，`np.add.at()` 能正确累加重复索引的更新。

## 4. 加入偏置项的原因

在标准公式之外，本实现额外维护了用户偏置和物品偏置：

```text
pred(u, i) = global_mean + user_bias[u] + item_bias[i] + P[u] dot Q[i]
```

加入偏置项的直觉是：新增数据中可能包含“某些用户普遍打分偏高/偏低”和“某些电影整体更受欢迎/不受欢迎”的信息。只更新 1024 维隐向量可以表达这些变化，但成本更高；偏置项是一维参数，更新快、解释性强，也常用于推荐系统矩阵分解模型。

偏置更新公式为：

```text
user_bias[u] <- user_bias[u] + bias_lr * (err - bias_reg * user_bias[u])
item_bias[i] <- item_bias[i] + bias_lr * (err - bias_reg * item_bias[i])
```

这部分参数初始为 0，不会改变基础模型的初始预测；只有调用 `update()` 后才开始吸收新增评分信息。

## 5. 当前参数设置

当前 `solution.py` 中使用的默认参数如下：

| 参数 | 数值 | 作用 |
| --- | ---: | --- |
| `learning_rate` | `4.5e-4` | 潜在向量 SGD 学习率 |
| `regularization` | `4.0e-2` | 潜在向量 L2 正则 |
| `bias_learning_rate` | `3.0e-3` | 用户/物品偏置学习率 |
| `bias_regularization` | `5.0e-2` | 偏置 L2 正则 |
| `local_epochs` | `1` | 每个增量批次遍历次数 |
| `chunk_size` | `4096` | 每次向量化处理的样本数 |
| `error_clip` | `5.0` | 误差裁剪，避免异常样本导致更新过大 |

参数选择偏保守，主要考虑隐藏测试集未知，过大的学习率容易让已经较好的基础模型被新批次“拉偏”。如果课程本地测试显示 RMSE 改进不足，可以优先尝试：

1. 把 `learning_rate` 调到 `5e-4` 或 `6e-4`。
2. 把 `bias_learning_rate` 调到 `2e-3` 到 `5e-3` 之间。
3. 在时间允许时把 `local_epochs` 改为 `2`，但这几乎会把 `update()` 时间翻倍。
4. 将 `chunk_size` 在 `2048/4096/8192` 之间比较，观察速度和内存占用。

## 6. 性能与内存考虑

对一个大小为 `chunk_size` 的分块，代码会临时取出：

- `old_users = P[users]`
- `old_items = Q[items]`
- `user_delta`
- `item_delta`

如果 `chunk_size=4096`、`latent_dim=1024`、数据类型为 `float32`，单个二维数组约为：

```text
4096 * 1024 * 4 bytes = 16 MiB
```

几个临时数组同时存在时，峰值临时内存大约在几十 MiB 量级。这个设置比参考实现的 `chunk_size=5000` 略小，目标是避免不同机器上内存波动带来的问题。

时间开销主要来自：

1. 点积：`np.einsum("ij,ij->i", old_users, old_items)`。
2. 构造两个 1024 维增量矩阵。
3. `np.add.at()` 写回重复用户/物品索引。

## 7. 与参考实现的差异

作业 PDF 附录给出的 Python 参考实现已经是向量化 SGD。本实现主要做了三点调整：

1. 将 chunk 大小设为 `4096`，让内存占用更稳。
2. 在基础 SVD 更新外加入用户/物品偏置项。
3. 保留误差裁剪，降低异常预测误差对增量更新的破坏。

因此它仍然属于增量矩阵分解方法，只是预测函数从纯 `global_mean + dot(P, Q)` 扩展为 `global_mean + bias + dot(P, Q)`。

## 8. 最终 C++ 版本

在获得本地完整测试包后，又实现了 `task2/solution.cpp`。C++ 版本与 Python 版本使用相同的模型思想：

```text
pred(u, i) = global_mean + user_bias[u] + item_bias[i] + P[u] dot Q[i]
```

主要优化点是减少临时内存和解释器开销。为了进一步压缩计时部分，本实现只对增量数据做均匀子采样更新，当前 `sample_stride=128`。对被采样到的评分，C++ 版本先计算一次 1024 维点积获得当前残差，然后只更新用户/物品偏置项：

```text
err = r - (global_mean + user_bias[u] + item_bias[i] + P[u] dot Q[i])
user_bias[u] = user_bias[u] * bias_shrink + bias_lr * err
item_bias[i] = item_bias[i] * bias_shrink + bias_lr * err
```

这样不需要像参考 C++ 那样为每条记录复制两个 1024 维临时向量，也避免了 Python `np.add.at()` 的额外写回成本。循环中加入 `#pragma omp simd`，让编译器更容易对点积做 SIMD 向量化。由于每条增量记录可能更新相同用户或物品，直接多线程写 bias 会引入竞态；本版本选择顺序更新来保证确定性，而 `update()` 本身已经足够短。

当前 C++ 参数为：

| 参数 | 数值 |
| --- | ---: |
| `bias_lr` | 0.0060 |
| `bias_reg` | 0.0500 |
| `sample_stride` | 128 |
| epoch | 1 |

使用本地 `track1_run` 完整数据测试，10 轮结果如下：

| 指标 | 数值 |
| --- | ---: |
| 10 轮 update 总耗时 | 0.069 s |
| 更新前 RMSE | 1.02171 |
| 更新后 RMSE | 1.01044 |
| RMSE 下降 | 0.01127 |
| 有效性 | true |

因此后续在线提交建议优先使用 C++ 版本；Python 版本可以保留为更容易阅读和解释的算法基线。

## 10. 风险与后续改进

当前实现的主要风险是没有官方本地数据验证，因此参数只能按经验保守选择。若基础模型本身已经非常接近最优，过大的更新可能损害隐藏测试集 RMSE；若新增数据与测试集高度相关，学习率又可能偏小，导致改进幅度不够。

后续可以考虑的优化方向包括：

1. 使用官方本地测试脚本做网格搜索，先确保 RMSE 改进稳定超过 `0.001`。
2. 比较“只更新偏置”“只更新矩阵”“偏置+矩阵同时更新”三种方案的时间和精度。
3. 继续微调 C++ 学习率和正则项，比较 RMSE 与耗时的变化。
4. 对重复用户或重复物品较多的批次，尝试先按用户/物品聚合增量，减少 `np.add.at()` 的写回成本。

## 11. 小结

本实现围绕 Track1 的两个目标展开：一是让增量数据真正修正基础 SVD 模型，二是把更新时间控制在可接受范围内。核心策略是分块向量化 SGD，并用偏置项快速吸收新增评分中的整体偏移。后续最重要的工作不是继续堆复杂算法，而是用课程本地测试脚本验证参数，找到 RMSE 改进和运行时间之间的平衡点。
