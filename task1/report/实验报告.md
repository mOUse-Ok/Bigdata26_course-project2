## 3. 算法设计

### 3.1 全局均值模型

全局均值模型使用训练集中所有评分的平均值作为任意 `(u, i)` 的预测值：

```text
pred(u, i) = mu
```

该方法不区分用户和物品，只作为最低复杂度基线。

### 3.2 带收缩的用户/物品均值模型

在全局均值的基础上，分别估计用户偏好和物品受欢迎程度。为了避免低频用户或低频物品过拟合，对偏置项加入收缩：

```text
b_u = sum_{i in I(u)} (r_ui - mu) / (|I(u)| + alpha_u)
b_i = sum_{u in U(i)} (r_ui - mu) / (|U(i)| + alpha_i)
pred(u, i) = mu + b_u + b_i
```

实验中使用 `alpha_u = 20`，`alpha_i = 40`。对冷启动用户或冷启动物品，对应偏置项置为 0。

### 3.3 正则化偏置 SGD 模型

最终采用正则化用户/物品偏置模型。模型形式为：

```text
pred(u, i) = mu + b_u + b_i
```

优化目标为：

```text
min sum (r_ui - mu - b_u - b_i)^2 + lambda * (sum b_u^2 + sum b_i^2)
```

使用随机梯度下降训练。对每条评分记录，误差为：

```text
e_ui = r_ui - pred(u, i)
```

更新公式为：

```text
b_u <- b_u + eta * (e_ui - lambda * b_u)
b_i <- b_i + eta * (e_ui - lambda * b_i)
```

本次默认参数为：

| 参数 | 数值 |
| --- | ---: |
| 验证集比例 | 20% |
| 随机种子 | 42 |
| 训练轮数 | 35 |
| 学习率 eta | 0.007 |
| L2 正则 lambda | 0.05 |

预测时将评分裁剪到训练集评分范围 `[10, 100]`。若测试集中用户或物品未在训练集中出现，则使用全局均值加已知的另一侧偏置；若两侧均未知，则退化为全局均值。

## 4. 实验结果

从训练集中随机划分 20% 作为验证集，剩余 80% 用于训练。评价指标为 RMSE 和 MAE：

```text
RMSE = sqrt(mean((r - pred)^2))
MAE  = mean(abs(r - pred))
```

验证集实验结果如下：

| 模型 | RMSE | MAE | 训练时间 | 峰值内存 | 参数空间估计 |
| --- | ---: | ---: | ---: | ---: | ---: |
| 全局均值 | 20.7231 | 16.4563 | 0.002 s | 0.000 MiB | 0 B |
| 收缩用户/物品均值 | 17.8158 | 13.7209 | 0.066 s | 1.385 MiB | 72048 B |
| 正则化偏置 SGD | 17.2178 | 13.2119 | 6.607 s | 1.129 MiB | 72048 B |

最终使用全部训练集训练正则化偏置 SGD 模型，并生成测试集预测结果：

| 项目 | 数值 |
| --- | ---: |
| 最终模型训练时间 | 8.318 s |

## 5. 分析

全局均值模型只使用评分整体分布，无法刻画用户偏好差异和物品质量差异，因此 RMSE 最高。加入用户偏置和物品偏置后，模型可以表达“某些用户普遍打高分”和“某些物品普遍评分高”的规律，RMSE 从 20.7231 降到 17.8158。

正则化偏置 SGD 进一步通过迭代优化用户偏置和物品偏置，使二者在同一目标函数下共同拟合评分，验证集 RMSE 降到 17.2178，是本次实验中效果最好的模型。该模型的参数量只与用户数和物品数线性相关，最终模型参数空间估计约 75.6 KiB，训练和预测开销都较小。

测试集中存在 12 个训练集未出现用户和 647 个训练集未出现物品。偏置模型对冷启动情况较稳健：未知用户或未知物品可以自然退化为全局均值与已知一侧偏置的组合，因此不会因缺少交互历史而无法预测。

矩阵分解模型也已在代码中保留为可选模型，但在一次相同验证集拆分下，24 维、45 轮的矩阵分解模型训练时间超过 5 分钟且 RMSE 未优于偏置模型。考虑到本数据集用户数较少、物品较多且矩阵稀疏，本次初稿选择泛化更稳定、复现实验更快的偏置 SGD 模型作为最终提交结果。

## 7. 后续可改进方向

1. 对学习率、正则项和训练轮数进行更系统的网格搜索或交叉验证。
2. 引入隐式反馈特征，例如用户评分数量、物品评分数量等频次特征。
3. 在矩阵分解中加入更强正则、早停或归一化处理，以降低稀疏矩阵上的过拟合风险。
4. 若允许使用第三方库，可使用 NumPy 或 Spark MLlib 加速矩阵分解与参数搜索。
